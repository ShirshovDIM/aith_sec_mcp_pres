03 -- OWASP MCP Top 10

OWASP MCP Top 10 -- Обзор рисков

MCP01 Token Mismanagement

Хардкод токенов, отсутствие ротации, утечка секретов через контекст и логи. Critical

MCP02 Privilege Escalation

Дрейф прав, отсутствие least privilege, расширение scope без аудита. High

MCP03 Tool Poisoning

Подмена схем инструментов, отсутствие подписи и верификации целостности. Medium-High

MCP04 Supply Chain Attacks

Вредоносные зависимости, floating versions, отсутствие SBOM. High

MCP05 Command Injection

shell=True, eval(), exec() -- выполнение произвольных команд через пользовательский ввод. Critical

MCP06 Intent Flow Subversion

Перехват намерений через контекст, скрытые инструкции в ресурсах. High

MCP07 Insufficient Auth

Отсутствие mTLS, статические токены, no RBAC. High

MCP08 Lack of Audit

Нет централизованного логирования, отсутствие SIEM интеграции, нет correlation ID. Medium

MCP09 Shadow MCP Servers

Неконтролируемые инстансы MCP с default-конфигурациями. Medium

MCP10 Context Over-Sharing

Смешение контекстов, отсутствие TTL, утечка между сессиями/агентами. Medium

09 -- Ключевые находки

Ключевые находки по MCP-критериям

Критические проблемы

MCP03 -- Tool Poisoning (84% FAIL)
Ни один сервер не реализует криптографическую подпись схем. Это позволяет подмену tool definitions.

MCP Servers Archived (Score: 12)
7 из 9 критериев -- FAIL. SQL injection в PostgreSQL sub-server. Не использовать в production.

Excel MCP (Score: 36)
5 категорий FAIL: MCP02, MCP03, MCP06, MCP07, MCP10. Отсутствует авторизация и контроль намерений.

MCP06 -- Intent Flow (0% PASS)
Ни один сервер не реализует guardrail model для валидации намерений. 81% -- PARTIAL.

Положительные тренды

MCP01 -- Token Management (79% PASS)
Все серверы используют env vars для секретов. Нет хардкода credentials в production коде.

MCP05 -- Command Injection (74% PASS)
Отсутствие shell=True, eval(), exec() в runtime. Безопасные параметризованные запросы.

Codebase Memory (Score: 100)
Единственный сервер с идеальным результатом. 8 PASS, 2 N/A. Образец безопасной архитектуры.

Official серверы лучше Community
Средний score Official: 62 vs Community: 59. Серверы от крупных вендоров (GitHub, AWS, MongoDB) стабильно в топе.

11 -- Рекомендации

Рекомендации по улучшению

Немедленно
  • НЕ использовать Archived серверы
  • Пин версий зависимостей (убрать ^, ~, latest)
  • Реализовать подпись схем (MCP03)
  • Добавить audit logging
Краткосрочно
  • Scope-based access control (RBAC)
  • Structured logging + correlation IDs
  • Context isolation + TTL
  • Per-session authentication
Среднесрочно
  • Intent validation (guardrail models)
  • SIEM интеграция
  • Quarterly security audits
  • SBOM/CBOM для всех серверов

Приоритеты по MCP-критериям

P0: MCP03 Tool Poisoning

16/19 серверов FAIL. Внедрить Sigstore подпись или SHA256 хеширование для всех tool schemas.

P1: MCP06 Intent Flow

0% PASS. Добавить независимый intent classifier для проверки соответствия действий целям пользователя.

P2: MCP10 Context Isolation

84% PARTIAL. Реализовать namespace isolation, TTL для контекста и PII-фильтрацию.
